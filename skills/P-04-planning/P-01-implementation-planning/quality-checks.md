# Planning Quality Checks

Check:

1. the plan starts from approved requirements, approved architecture, and D-04 outputs
2. feature requirements and architectural requirements are listed with IDs plus plain-language summaries sourced from approved artifacts
3. every in-scope output doc is scheduled somewhere in the plan
4. checklist steps use checkboxes and substeps where the implementation slice needs them
5. any concrete code examples are pulled from D-04 output docs rather than invented in Planning
6. representative examples cover distinct implementation types rather than noisy copy-and-paste variants
7. no new design or contract content leaked into the plan
8. issues remain in the issues section rather than being silently converted into scope
9. step ordering reflects actual dependency order
10. the plan stays scheduling-only
